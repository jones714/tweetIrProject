 Problems We Faced

The Twitter API Key was not able to be retrieved in a timely matter. This resulted in the use of the Tweepy libraries. When pulling our data we had to create an algorithm that would allow our Tweepy client to refuse duplicate tweets and ignore retweets of the same tweet.

Future Optimizations

Once receiving an API Key the program will have stronger data to come up with a stronger analysis. In the future the project could implement more filters to make sure the tweets are better quality and only look for tweets that aren't just pictures. It could even implement some sort of analysis that includes the context of a photo in the overall analysis. The test data can be stronger to make the sentiment analysis more accurate. Machine learning is also a good future optimization as it will allow the analysis to better understand human sarcasm.
